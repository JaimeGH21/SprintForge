# SprintForge

Repositorio del proyecto SprintForge para la asignatura de Ingeniería del Software II.

## Descripción
Este repositorio gestiona el desarrollo del proyecto SprintForge. Se ha implementado un flujo de trabajo basado en ramas para asegurar la estabilidad del código y facilitar el desarrollo colaborativo entre los integrantes.

## Integrantes
* Jaime García
* Erika Macarro

## Estructura del Repositorio
Siguiendo los estándares de organización de proyectos Java:
* src/main/java/es/uclm/sprintforge/controller/ -> Lógica de negocio y control de reservas.
* src/main/java/es/uclm/sprintforge/interfaz/ -> Componentes de la interfaz de usuario y menús.
* docs/ -> Documentación técnica, diagramas y manuales del proyecto.
* pom.xml -> Archivo de configuración de Maven y gestión de dependencias.

## Metodología de Trabajo
El proyecto utiliza un sistema de control de versiones organizado por niveles:
1. master: Contiene la versión estable y verificada del software.
2. develop: Rama principal donde se integran las nuevas funcionalidades.
3. Ramas de tareas: Uso de ramas específicas para cada módulo (ej. interfaz, feature-reservas) antes de su integración final.

## Objetivo
El objetivo es desarrollar una aplicación funcional aplicando principios de Ingeniería del Software, gestión avanzada de repositorios en GitHub y resolución de conflictos en entornos de trabajo reales.
